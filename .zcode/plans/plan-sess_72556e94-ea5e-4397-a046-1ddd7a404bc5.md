## Root cause (from Khalti's own docs, attached)

The official sample request (docs line 86-118) sends:
```json
"customer_info": { "name": "Khalti Bahadur", "email": "...", "phone": "..." }
```

And the first validation error in the docs is `customer_info.name` required. Although `customer_info` is listed as "No" (optional), the **sandbox enforces `name`**.

Your current code fails because:
1. `profile.display_name` is `NULL` → `buyerName` is `undefined` → no name sent.
2. Even if a name were passed, the **email spread on the next line overwrites the entire `customer_info` object**, dropping `name` (verified by reading `khalti.ts` line 63-64 — two separate `customer_info` spreads).

## Fix (both changes, because either alone fails)

### 1. `src/lib/payments/khalti.ts` (lines 63-64)
Merge into a single `customer_info`:
```ts
...(params.buyerName || params.buyerEmail)
  ? { customer_info: {
      ...(params.buyerName ? { name: params.buyerName } : {}),
      ...(params.buyerEmail ? { email: params.buyerEmail } : {}),
    } }
  : {}),
```

### 2. `src/app/api/checkout/initiate/route.ts` (~line 96)
Provide a name so it's never undefined:
```ts
buyerName: profile?.display_name ?? user.email?.split("@")[0],
```

Resulting payload to Khalti: `customer_info: { name: "dangolraunak", email: "dangolraunak@gmail.com" }` — matches the docs sample exactly.

After this, `khaltiInitiate` returns a `payment_url` (`https://test-pay.khalti.com/?pidx=...`) which the client redirects to — the proper payment flow you described. (Verified earlier that `/epayment/initiate/` returns HTTP 200 with a real `pidx`.)
