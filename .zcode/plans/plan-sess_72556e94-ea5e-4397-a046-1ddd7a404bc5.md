# TikTok-Style Vertical Live Feed (per attached spec)

Goal: make `/` a full-screen, **live-only** vertical snap feed where **exactly one** stream at a time holds a LiveKit connection, Supabase Realtime channels, presence, polling, or timers. Move the existing grid homepage to `/browse` (keeps the `(shop)` Navbar). `/stream/[id]` is untouched.

## Architecture
```
src/app/page.tsx            (Server Component, root → uses root layout = NO Navbar)
└── StreamFeed              ("use client"; owns activeIndex + documentVisible)
    ├── inactive slide → static poster (no StreamView mounted)
    └── active slide → StreamView (active)
                    └── StreamRoom (active) → VideoStage (connect={active})
```
`StreamFeed` owns the single `activeStreamId`. Every resource-producing component receives `active`; inactive components clean up immediately.

### 3. `src/app/page.tsx` (NEW — Server Component, root)
- `export const dynamic = "force-dynamic"`.
- Cookie-aware server client → `getAuthenticatedUser()`; load `profile.display_name` only when authenticated.
- Query **live-only** with explicit relations (not `seller(*)`):
  ```ts
  .from("streams")
  .select(`*, seller:seller_id(id, display_name, is_verified), pinned_product:pinned_product_id(*)`)
  .eq("status", "live").order("created_at", { ascending: false })
  ```
- Do **not** derive one global role (it's per-stream in StreamFeed). Render:
  ```tsx
  <StreamFeed streams={streams} viewerId={user?.id ?? null} viewerName={profile?.display_name ?? null} />
  ```
- Empty state: full-screen black "No live streams right now" + CTA → `/browse`.

### 4. `src/components/StreamFeed.tsx` (NEW — "use client")
- State: `activeIndex` (0), `documentVisible` (true); stable ref collection for slides.
- Effective active: `isActive = documentVisible && index === activeIndex`.
- Container: `h-dvh overflow-y-auto overscroll-y-contain snap-y snap-mandatory bg-black`; each slide `h-dvh snap-start snap-always relative` (dvh for mobile chrome).
- **One IntersectionObserver** (`{root, threshold:[0.25,0.5,0.6,0.75,1]}`): among all intersecting entries pick **highest ratio**, switch `activeIndex` only when ratio ≥ 0.6; ignore callback order; disconnect on cleanup.
- **visibilitychange**: hidden → `documentVisible=false` (releases LiveKit/Realtime/presence/polling); visible → reactivate centered slide.
- Active slide renders `<StreamView active ...>`; inactive → lightweight poster (title, seller, thumbnail/black). Never mount `StreamView` for inactive slides. Per-stream role: `viewerId && stream.seller_id === viewerId ? "seller" : "viewer"`.
- Fixed top-right grid button → `router.push("/browse")`, label "Browse streams".

### 5. `src/components/stream/StreamView.tsx` (EDIT)
- Add `active?: boolean` (default `true`). Normalize `const isActive = active ?? true;`. Pass into `<StreamRoom active={isActive}>`.
- Every resource-allocating effect begins `if (!isActive) return;` with `isActive` in deps, verified cleanup for: chat `postgres_changes`, stream/pinned-product changes, reactions, **presence channel (untrack + remove)**, purchase-ticker polling, rank polling, reaction-batch timers, delayed UI timers, window/document listeners. On deactivate, reset stale ephemeral state (bullet comments, pending reactions, ticker data).

### 6. `src/components/StreamRoom.tsx` (EDIT)
- Add `active?: boolean`; normalize. Token effect:
  ```ts
  useEffect(() => {
    if (!isActive) { setToken(null); return; }
    const controller = new AbortController();
    void fetchToken(controller.signal);
    return () => { controller.abort(); setToken(null); };
  }, [isActive, stream.id, role]);
  ```
  Ignore `AbortError`; prevent stale responses from setting state; clear token on inactive/cleanup.
- Mount video only when `isActive && token`: `<VideoStage active token={token} />` else `<StaticPoster />`. (Conditional mount = primary guarantee.)

### 7. `src/components/VideoStage.tsx` (EDIT)
- Add `active?: boolean`; replace `connect={true}` → `connect={active ?? true}` (second defensive layer). Verify room disconnects/disposes on unmount.

### 8. `/stream/[id]/page.tsx` — NO change
Default `active=true` preserves immediate token/LiveKit/chat/presence/ticker/rank behavior.

## Tests (vitest)
- **StreamFeed**: first slide active; highest ratio wins; <0.6 no switch; only one `StreamView` mounted; ownership → "seller" only for own stream; hidden doc deactivates; grid button → `/browse`.
- **StreamRoom**: inactive → no token request; activate → one request; deactivate aborts in-flight; stale response can't set token; `VideoStage` mounts only when active+token; deactivate removes `VideoStage` + clears token.
- **StreamView**: inactive → no Supabase channels; activate → opens expected channels once; deactivate removes every channel/timer; presence untracked on cleanup.
